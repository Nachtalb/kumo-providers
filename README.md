# kumo-providers

The **community provider registry** for [Kumo](https://github.com/Nachtalb/kumo)
— the manga reader. Each provider is a small, sandboxed **Lua 5.4** script that
teaches Kumo how to browse and read one manga source.

This repo is the single source of truth that the Kumo **app** and **server**
consume for **sha-verified auto-update**: they fetch [`index.json`](index.json),
compare the recorded `sha256` of every `provider.lua` (and its icon) against what
they already have, and pull only what changed. Because every artifact is hashed,
a tampered or truncated download is rejected before it ever runs.

> **New providers install disabled by default.** After a registry sync, a freshly
> added source shows up in Settings → Sources switched **off**; users opt in
> explicitly. Nothing new runs behind your back.

## Layout

```
providers/
  <id>/
    provider.lua   # the Lua 5.4 script (metadata header + contract functions)
    icon.avif      # square source icon
    readme.md      # short human description
index.json         # generated manifest — DO NOT hand-edit
scripts/
  build-index.mjs  # regenerates index.json (Node, zero deps)
docs/
  authoring.md     # how to write a provider
.github/workflows/
  validate.yml     # CI: regenerate index, fail if stale, (todo) sandbox-validate
```

## Contributing a provider

1. Read [`docs/authoring.md`](docs/authoring.md) — the metadata header, the
   contract functions, the host API, and the output schemas.
2. Create `providers/<your-id>/` with `provider.lua`, `icon.avif`, `readme.md`.
   The directory name **must** equal the script's `-- @id`.
3. Regenerate the manifest and commit it alongside your provider:
   ```bash
   node scripts/build-index.mjs
   git add providers/<your-id> index.json
   ```
4. Open a PR. CI regenerates `index.json` and fails if yours is out of date.

## The `index.json` manifest

Generated by `scripts/build-index.mjs`. Each entry:

| field                   | meaning                                                        |
| ----------------------- | ------------------------------------------------------------- |
| `id`                    | provider id (equals the directory name and `-- @id`)          |
| `name`                  | display name (`-- @name`)                                      |
| `version`               | semver (`-- @version`)                                         |
| `langs`                 | array of language codes (`-- @langs en,de`)                   |
| `nsfw`                  | boolean (`-- @nsfw`)                                           |
| `requires_verification` | boolean — true if the script declares `-- @verify`            |
| `created` / `updated`   | ISO dates from git history (else generation time)             |
| `url`                   | path to the script, e.g. `providers/weebcentral/provider.lua` |
| `sha256`                | hex sha256 of `provider.lua`                                   |
| `icon` / `icon_sha256`  | path + hex sha256 of the icon (null if none)                  |
| `ua_hint`               | preferred UA family (`-- @ua chrome`)                          |

The manifest is deliberately **byte-stable** across runs (no volatile
timestamps) so CI can trust `git diff --exit-code index.json`.
